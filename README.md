### PURPOSE
This project aims to create an E-commerce website to provide a satisfying consumer and seller experience.

  In this modern fast-paced and busy world, E-commerce platforms provide a very convenient way for people across the world to get their desired products at their doorstep. 

Hence, E-commerce platforms must be fast, convenient and efficient to cater to the needs of both the buyer and seller.

### SOFTWARE REQUIREMENTS
#### System Architecture:

The e-commerce website will be built using a client-server architecture.
The server will host the website's database and business logic, while
the client will be a web browser.

#### Programming Languages and Technologies:

❖ Frontend: Eg- HTML5, CSS3, JavaScript
❖ Backend: Eg- JAVA
❖ Database: Eg-Mongo-DB

#### Third-Party Integrations:

 Google Maps for shipping, social media sharing, and email notifications.

### HARDWARE REQUIREMENTS
#### Server Hardware:

The server hosting the e-commerce website should meet the following
requirements:

❖ Processor: Eg-CPU or higher
❖ RAM: Eg-8GB or more
❖ Storage: Eg-SSD with sufficient capacity
❖ Network: Eg-High-speed internet connection

#### Client Hardware:

The client hardware requirements are minimal, including any device
with a modern web browser capable of rendering HTML5 and CSS3.

### FUNCTIONAL REQUIREMENTS
#### User Registration and Authentication:
❖ Users should be able to register with a unique username and password.
❖ Users can log in and log out of their accounts.
❖ Password reset functionality via email.

#### Product Catalogue:
❖ Display products in various categories with images, descriptions, and prices.
❖ Users can search for products using keywords and filters.
❖ Product reviews and ratings.

#### Shopping Cart:
❖ Users can add and remove items from their shopping cart.
❖ Calculate and display the total order amount.
❖ Apply discounts or promo codes if applicable.

#### Payment Processing
❖ Securely handle payment information.
❖ Integration with Stripe for credit card payments.
❖ Provide payment confirmation to users.

#### Order Management
❖ Users can view their order history and order status.
❖ Admins can manage orders, update order status, and handle returns/refunds.

### NON FUNCTIONAL REQUIREMENTS

#### Performance
❖ The website should load quickly and be responsive.
❖ Handle a high volume of concurrent users.
❖ Database queries should be optimized for speed.

#### Security
❖ Protect against mongodb injection and other common web security threats.
❖ Regular security audits and updates.

#### Scalability
❖ The system should be scalable to accommodate future growth.
❖ Easy integration with CDN services for content delivery.

#### Usability
❖ User-friendly interface with a responsive design for mobile and desktop.
❖ Accessibility features for users with disabilities.

#### Availability
❖ The website should have high uptime, with minimal downtime for maintenance.
❖ Implement backup and disaster recovery procedures.
